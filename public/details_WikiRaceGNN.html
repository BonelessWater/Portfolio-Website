Inspiration

The Wikipedia game (finding the shortest path between two articles) has always fascinated me as a practical application of graph theory. However, traditional breadth-first search becomes computationally expensive at scale. I wanted to explore whether Graph Neural Networks could learn the structure of Wikipedia's link graph and use that knowledge to navigate more intelligently than conventional algorithms.

Technical Innovation

This project implements advanced graph traversal algorithms using GraphSAGE (Graph Sample and Aggregate) to navigate Wikipedia's massive link graph of 7 million pages. The system combines state-of-the-art deep learning with sophisticated search strategies including Parallel Beam Search, Bidirectional Guided Search, and Hybrid approaches. The GNN learns embeddings that capture the semantic relationships between Wikipedia articles, enabling the algorithm to make smarter decisions about which links to explore.

System Architecture

The architecture consists of several key components working in concert. The data pipeline generates graph representations from Wikipedia's edge lists, creating adjacency matrices, node embeddings, and mapping structures. The GraphSAGE model trains on this data to learn meaningful representations of the graph structure. The traversal engine implements multiple search strategies: Beam Search explores multiple promising paths simultaneously, Bidirectional Search works from both start and end nodes, and the Hybrid approach combines both techniques. An adaptive selection mechanism automatically chooses the most appropriate strategy based on estimated node distance.

Performance Optimization

Through careful optimization, the GNN-guided search typically explores 30-70% fewer nodes than traditional BFS while still finding optimal or near-optimal paths. The system employs selective caching to manage memory efficiently even with millions of nodes, and the core logic is designed with parallelization in mind for future scaling. The interactive Flask dashboard provides real-time visualization of the traversal process, allowing users to compare GNN performance against BFS, animate search steps, and analyze detailed statistics including path length, nodes explored, and execution time.

This project taught me how to apply cutting-edge machine learning techniques to classic computer science problems. Working with graphs at this scale (7 million nodes) presented unique engineering challenges around memory management, data structure design, and computational efficiency. I gained deep experience with Graph Neural Networks, learning how message-passing architectures can capture complex relational patterns.
